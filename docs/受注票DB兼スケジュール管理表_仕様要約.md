# 受注票DB兼スケジュール管理表 仕様要約

## 1. 目的
- Excel/VBA で受注情報・工程日程・スケジュール表示・受注表発行・履歴管理を一元化する。
- 製番キーで検索/編集できる運用を前提とする。
- 完了/中止/論理削除をフラグ管理し、編集履歴に Windows ID を残す。
- 受注表は PDF ではなく Excel（`.xlsx`）で発行する。
- 将来、Microsoft Forms を入口として取り込むハイブリッド運用を検討する。

## 2. 基本構成
- 操作ブック: `受注表管理.xlsm`
- DBブック: `受注票_DB.xlsx`
- DBパス: `DB_PATH.txt` または設定シート（コード直書き禁止）
- スケジュール表示: `工程表` シート
- 受注表テンプレート: 操作ブック内 `受注票` シート
- VBAソース: `.bas` は Shift-JIS 推奨

## 3. 環境方針
- フォント: Noto Sans JP（代替: メイリオ）
- Excel: デスクトップ版前提
- UserForm: `.frm` 直接インポートではなく自動生成マクロ方式

## 4. DBシート構成
`受注票_DB.xlsx` に以下を配置:
- `DB_受注`
- `DB_履歴`
- `DB_発行ログ`
- `M_注文区分`, `M_取引先`, `M_営業担当`, `M_部署`, `M_工程`, `M_略称`, `M_設定`

## 5. 受注データ要件
- 一意キー: 製番
- 必須: 注文区分、受注日、納期、営業担当、該当部署（1つ以上）
- 任意入力: 取引先名、担当者名、機種番号、機種名、仕様、金額
- 入力規制:
  - 見積No/注文No: 半角英数字記号
  - 金額: 数値のみ（税抜、カンマ整形）
  - 日付項目: カレンダー入力

## 6. 製番ルール
- 形式: `YY-営業所区分+4桁連番-2桁枝番`
- 例: `26-E0100-01`
- 営業所区分: `B / E / Q`
- 正式採番は「新規登録時にDBロック取得後」に実施し、直前再確認で重複防止。

## 7. 登録/編集モード
- モード: 新規 / 編集 / 複製
- 新規:
  - 製番必須
  - DB重複不可
  - ロック取得後に再重複確認
- 編集:
  - 製番変更不可
  - 更新日時チェックで他者更新を警告

## 8. フォーム分離
- `受注情報入力フォーム` と `工程情報入力フォーム` を分離する。
- 受注フォーム:
  - 採番、製番読込、詳細検索、部署チェック
  - 「現在のモード」は大きいフォント
- 工程フォーム:
  - 製番検索で既存工程読込
  - 該当部署外の工程はグレーアウトして入力不可

## 9. 検索方針
- 製番読込: 製番既知時の直接読込
- 詳細検索: 複数条件での絞り込み（複数ヒット前提）
- 条件例: 製番、注文No、見積No、顧客、担当者、日付範囲、金額範囲、ステータス各種

## 10. 工程と部署の連動
- 機械設計→機械設計
- 資材→資材
- ファームウェア/アプリケーション→ソフト設計
- 製造/サービス→製造入力可
- デバッグ/検査→常時入力可
- 品証/伝票発行→工程日程なし

## 11. 工程入力ルール
- 開始・終了とも空欄: 登録可
- 片側のみ入力: エラー
- 開始 > 終了: エラー
- 工程終了 > 納期: 警告して登録可
- 該当部署ありで工程未入力: 警告して登録可

## 12. 工程未入力抽出（検討中）
- 未完了/未中止/未削除を対象に未入力工程を一覧化
- 表示項目案: 部署、製番、納期、未入力工程、状態、備考
- 優先ソート: 該当部署 → 納期昇順 → 製番順
- 判定区分: 未入力、片側入力、日付逆転、納期超過
- サービス工程を必須対象にするかは未確定

## 13. 工程表（週次）
- `工程表` シートに週単位表示（週開始: 月曜）
- 表示範囲: 当週月曜 ～ DB内最長納期
- 並び順: 納期昇順
- 1製番2行:
  - 上段: ハード（機械設計/資材/製造/デバッグ/検査）
  - 下段: ソフト（ソフト設計/デバッグ/検査）
- 開始週に略称を表示し、重複開始は警告色
- 完了/中止/削除は通常表示から除外

## 14. ステータス
- 通常 / 完了 / 中止 / 論理削除
- DBフラグ: 完了フラグ、中止フラグ、削除フラグ

## 15. 履歴管理
`DB_履歴` に以下を記録:
- 履歴ID、対象ID、製番、操作区分、変更日時、Windows ID、変更項目、変更前後
- 編集時は変更項目のみ記録
- 運用目安:
  - 30,000行超で古い履歴を `HistoryArchive` に退避
  - DB内は直近20,000行程度を維持

## 16. 排他制御
- 書込系処理のみ短時間ロック
- 検索/工程表更新はロック不要
- 安全策: 書込直前再読込、更新日時チェック、タイムアウト、所有者表示、バックアップ

## 17. 書込位置の確定ロジック
- `最終行+1` 依存を廃止
- 2行目以降を先頭から走査し「最初の空行」に書込む
- 既存空白行は `CompactDbSheetsNow` で整理

## 18. 受注表発行
- 出力: `.xlsx`
- 保存先: DBフォルダ配下 `受注表Excel`
- 命名: `受注表_製番_取引先名_yyyymmdd.xlsx`
- 発行ログ: `DB_発行ログ`
- 押印済み取込（将来）:
  - 管理ブック側の取込処理で完了フラグ更新

## 19. Forms連携方針
- Forms は新規受付の入口として利用可
- 採番・正式登録・工程・履歴・発行は Excel/VBA 側で実施
- 完全クラウド化は現時点では見送り

## 21. 継続開発への実装原則
- DBは分離ブック運用、パスは設定で取得
- フォームは「受注」「工程」を分離
- 製番採番はロック後に確定
- 工程表は1製番2行、週次表示
- 書込は「最初の空行」方式
- 履歴は変更項目単位で保存しアーカイブ運用
- 受注表はExcel出力
- 文字コードは Shift-JIS、フォントは Noto Sans JP（代替メイリオ）
