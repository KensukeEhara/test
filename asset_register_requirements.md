# 資産管理台帳アプリ 要件定義（Excel .xlsm）

最終更新日: 2026-05-20

## 1. 基本方針
- 形式: Excel（`.xlsm`）
- 会社名（印刷固定文言）: **メディカテック株式会社**
- 目的: 固定資産/リース資産の台帳管理、採番、印刷、監査可能な編集履歴管理

## 2. 入力項目
| 項目 | 必須 | 入力方式 | 備考 |
|---|---|---|---|
| 資産種別 | 必須 | プルダウン | 種別マスタ参照 |
| 購入orリース | 必須 | プルダウン | 候補: 購入 / リース |
| 設置場所 | 必須 | プルダウン | 設置場所マスタ参照 |
| 名称 | 必須 | テキスト | |
| 型式 | 必須 | テキスト | |
| 取得年月日 | 必須 | 日付 | YYYY/MM/DD |
| 取得額 | 任意 | 数値 | 0以上 |
| 耐用年数 | 任意 | 数値 | 1以上整数 |
| 償却方法 | 任意 | プルダウン | 定額法 / 定率法 / 非償却 / 即時償却 |
| 償却率 | 任意 | パーセント | ％入力（例: 25%） |
| 管理番号 | 任意 | テキスト | 自由入力、重複許容 |
| 備考 | 任意 | テキスト | |
| 資産番号 | 自動 | 自動採番 | 手入力不可 |
| 状態 | 任意 | プルダウン | 使用中 / 廃棄 / 売却 / リース返却 |

## 4. 自動採番仕様

### 4.1 フォーマット
`LLTTYYSSS`（内部保存）
- `LL`: 設置場所コード2桁
- `TT`: 種別コード2桁
- `YY`: 取得年月日の西暦下2桁
- `SSS`: 連番3桁（001〜999）

### 4.2 採番単位
連番は **(LL, TT, YY) 単位** で管理する。  
（候補2を採用）

### 4.3 採番ルール
- 新規登録時に採番する。
- 対象キー内の最大連番 +1 を採用する。
- 初回は `001`。
- 欠番は詰めない。
- いかなる理由でも採番済み番号の再利用は禁止。

## 5. 編集時処理（確定方針）

### 5.1 資産番号固定
- 一度採番した資産番号は**不変**。
- 設置場所/資産種別/取得年月日を後から編集しても、資産番号は変更しない。

### 5.2 重要項目編集時
重要項目: 設置場所、資産種別、取得年月日、名称、型式

処理:
1. 編集確定時に警告表示: 「資産番号は変更されません。履歴を記録します。」
2. 変更理由入力を必須化。
3. 履歴記録後に保存。

### 5.3 監査ログ（変更履歴）
変更履歴シートに以下を記録:
- 変更日時
- 編集者名（Excel/Windowsユーザー）
- **編集者Windows ID**（例: `DOMAIN\\user` または `user`）
- 資産番号
- 対象項目
- 変更前
- 変更後
- 変更理由

### 5.4 削除方針
- 物理削除は禁止。
- 論理削除（状態変更）で管理する。

## 6. 印刷仕様（初版）
- 一覧で選択した**単一レコードのみ**印刷可。
- 複数レコード印刷は不可。
- 印刷項目:
  - 社名: メディカテック株式会社
  - 資産番号
- デザイン詳細は後続フェーズで調整。

## 7. シート構成（推奨）
- 台帳
- マスタ_設置場所
- マスタ_種別
- マスタ_償却方法
- 採番管理
- 変更履歴
- 印刷
- 設定

## 8. 入力制御・保護
- 必須未入力時は保存不可。
- 資産番号列はロックし、手動編集不可。
- 日付/数値/％形式の入力規則を設定。
- 重要マスタ・数式セルはシート保護。

## 9. 未確定（実装前に要判断）
- 連番上限 `999` 到達時の扱い（エラー停止/年度切替運用など）
- Windows ID取得方法の優先順位（`WScript.Network.UserName`、`Environ("USERNAME")` 等）
- 編集理由の最小文字数（例: 5文字以上）
- 履歴閲覧権限（全員閲覧可 or 管理者限定）
